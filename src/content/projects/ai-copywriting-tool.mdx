## Overview

A SaaS tool that enables marketers to generate, refine, and manage high-converting marketing copy using AI — across emails, landing pages, ad copy, and social media.

## Context

Content marketing teams spend an average of 3–5 hours writing a single blog post or landing page. Small teams without dedicated copywriters struggle to maintain consistent output quality. Existing AI writing tools at the time were generic and lacked marketing-specific frameworks.

## Problem

- Marketers needed faster content creation without sacrificing quality
- Existing AI tools generated generic text without marketing frameworks (AIDA, PAS, etc.)
- No workflow for team collaboration, version history, or brand voice consistency
- No integration with common marketing platforms

## My Role

As **Product Manager**, I conducted user interviews with 20+ marketers, defined the product positioning, and built the feature roadmap. As **Full-stack Developer**, I built the entire application — from the AI prompt engineering layer to the billing system.

## Product Decisions

1. **Framework-based generation**: Instead of free-form text generation, each template uses proven copywriting frameworks (AIDA, PAS, BAB) to produce structured, high-converting copy.
2. **Brand voice profiles**: Users define their brand voice (tone, vocabulary, dos/don'ts), and the AI adapts output accordingly.
3. **Freemium model**: 5 free generations per month, then tiered pricing via Stripe subscriptions.

## UX Flow

1. User selects a content type (email, landing page, social post, ad copy)
2. Fills in a brief (product, audience, goal, tone)
3. AI generates 3 variations using selected copywriting framework
4. User edits, refines, or regenerates
5. Final copy saved to project folder, exportable to clipboard or markdown

## Technical Implementation

- **AI Layer**: OpenAI GPT-4 API with custom system prompts per content type and framework
- **Frontend**: Next.js with optimistic UI updates for fast perceived performance
- **Database**: PostgreSQL via Prisma for user data, projects, and generation history
- **Billing**: Stripe Checkout + Webhooks for subscription management
- **Rate Limiting**: Redis-based rate limiting to prevent abuse on free tier
- **Auth**: NextAuth.js with Google and email login

## Result

- **500+ users** signed up in the first 30 days after ProductHunt launch
- **15% free-to-paid conversion** rate (industry average is ~3–5%)
- Users reported **60% faster** content creation workflow
- Average session length: **12 minutes** (high engagement for a SaaS tool)

## Reflection

Prompt engineering was the most impactful investment. Spending 2 weeks refining system prompts with real marketers resulted in dramatically better output quality than the initial generic prompts. I also learned that showing users the "framework" behind the generation (e.g., "This uses the AIDA framework") significantly increased trust and perceived value — users felt they were learning, not just outsourcing.
