## Overview

A modern real estate marketplace that combines interactive map-based property search with a streamlined listing management system for agents and property owners.

## Context

The client operated a traditional real estate brokerage and wanted to build a digital-first platform to compete with Zillow and Realtor.com in a regional market. They needed both a consumer-facing search experience and an agent-facing listing management tool.

## Problem

- Their existing website was a static WordPress site with no search functionality
- Agents managed listings via spreadsheets and email
- No mobile-optimized experience for buyers browsing on phones
- Competitors offered map-based search that their site lacked

## My Role

I defined the **product strategy** from competitive analysis through feature prioritization. As **Frontend Lead**, I designed and built the interactive map search experience. As **Full-stack Developer**, I built the API layer, database schema, and agent dashboard.

## Product Decisions

1. **Map-first search**: Research showed that 78% of home buyers start with location — so the map is the primary search interface, not a list.
2. **Saved searches with alerts**: Automated email notifications for new listings matching saved criteria to drive repeat engagement.
3. **Agent dashboard**: A simple CRM-like interface for agents to manage listings, track inquiries, and respond to buyers.

## UX Flow

1. Buyer opens the platform and sees a full-screen interactive map
2. Filters (price, bedrooms, type) dynamically update map markers
3. Clicking a marker shows a property card with key details
4. Full property page with gallery, details, and contact form
5. Agents receive inquiries in their dashboard and can respond directly

## Technical Implementation

- **Map**: Mapbox GL JS with custom marker clustering and dynamic filtering
- **Frontend**: Next.js with ISR for property pages (fast loads, SEO-friendly)
- **API**: Next.js API routes with Prisma ORM
- **Database**: PostgreSQL with PostGIS extension for geospatial queries
- **Image handling**: Cloudinary for optimization and responsive image delivery
- **Auth**: NextAuth.js with role-based access (buyer vs. agent)

## Result

- **40%** increase in property inquiries within the first quarter
- Average session duration **2.5x longer** than the old site
- Agents reduced listing management time by **60%**
- Mobile traffic increased from 30% to **65%** of total visits

## Reflection

Building the geospatial search was the most technically challenging part — optimizing Mapbox performance with thousands of markers required custom clustering logic and viewport-based data loading. On the product side, I learned that the agent experience is just as important as the buyer experience — if agents don't use the tool, listings dry up and the platform fails.
