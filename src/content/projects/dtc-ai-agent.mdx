## 项目背景

DTC 商家在售前咨询、订单查询、优惠券使用和售后政策解释上会遇到大量重复问题。传统 FAQ 能覆盖简单问题，但很难处理上下文、订单信息和多轮追问，也缺少对高风险场景的边界控制。

这个项目的目标是设计一个可演示的 AI 客服与导购 MVP：用户可以像聊天一样提出问题，系统根据意图选择政策问答、商品推荐、订单查询、优惠券发放或投诉升级等流程。

## 产品设计

- 将用户问题拆分为商品推荐、订单查询、优惠券、政策问答、投诉升级等意图。
- 设计 Agent 流程：意图识别、上下文抽取、工具调用、风险校验、回复生成。
- 为订单查询加入订单号和邮箱双重验证，避免直接暴露用户订单信息。
- 对退款、医疗功效、缺货商品和低置信度回答设置 guardrails，并在高风险场景转人工。

## 原型实现

后端使用 FastAPI 与 LangGraph 搭建 Agent 流程，MCP Tools 负责商品搜索、订单查询、政策检索、优惠券发放和升级记录。前端使用 React / Vite 构建聊天界面，并通过 SSE 展示流式响应。

项目同时保留了 mock 商家数据和分析看板，用于观察对话总量、意图分布、升级率、优惠券触发、热门问题和推荐商品。

## 复盘

这个项目最有价值的部分不是“让模型回答问题”，而是把模型放进一个可控业务流程里。真正需要设计的是工具边界、信息校验、异常处理和人工升级，而不是单纯追求回答更像人。
